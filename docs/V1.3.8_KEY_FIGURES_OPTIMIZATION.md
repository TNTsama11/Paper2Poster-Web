# v1.3.8 Key Figures 优化详解

## 📋 改进概述

在横向学术海报模板（`academic_panels_landscape.html`）中，左侧 **Key Figures** 面板现在至少显示 **2 张图片**，相比之前的单张展示，显著提升了信息密度和视觉吸引力。

---

## 🎯 改进动机

### 之前的问题
- ❌ 左侧 Key Figure 区域只显示 1 张图片
- ❌ 对于有多个重要结果的论文，无法同时展示
- ❌ 左侧栏空间利用不充分，存在大量留白
- ❌ 视觉冲击力不足，信息密度低

### 改进后的优势
- ✅ 至少显示 2 张关键图片，充分利用空间
- ✅ 可同时展示多个核心实验结果或架构图
- ✅ 增强海报的视觉吸引力
- ✅ 符合学术会议海报的信息密度要求

---

## 🔧 技术实现

### 智能选择策略

```jinja2
{# 1. 优先从 results 图片中选择 #}
{% if result_figures %}
    {% set sorted_results = result_figures | sort(attribute='priority', reverse=True) %}
    
    {# 1.1 如果 results 图片 >= 2 张 #}
    {% if sorted_results | length >= 2 %}
        {% set key_figures = sorted_results[:2] %}
    
    {# 1.2 如果 results 图片 < 2 张，从所有图片补充 #}
    {% else %}
        {% set all_sorted = poster.figures | sort(attribute='priority', reverse=True) %}
        {% set key_figures = sorted_results + (all_sorted | reject('in', sorted_results) | list)[:2 - (sorted_results | length)] %}
    {% endif %}

{# 2. 如果没有 results 图片，从所有图片中选择 #}
{% elif poster.figures %}
    {% set sorted_figs = poster.figures | sort(attribute='priority', reverse=True) %}
    {% set key_figures = sorted_figs[:2] %}

{# 3. 后备方案 #}
{% else %}
    {% set key_figures = [] %}
{% endif %}
```

### 选择优先级

1. **第一优先级**: `placement='results'` + 高 `priority` 的前 2 张
2. **第二优先级**: 如果不足 2 张，从所有图片中按 `priority` 补充
3. **第三优先级**: 如果总图片 < 2 张，显示所有可用图片

## 🎨 视觉优化

### CSS 自适应
- 每张图片的 `.figure-container` 都有独立的标题和说明
- 自动调整图片尺寸和间距，保持整体美观
- 支持不同宽高比的图片（架构图、对比图、图表等）

### 类型标签
- 每张图片都显示类型标签（`result`, `architecture`, `chart` 等）
- 彩色标签，方便快速识别图片类型

## 📋 使用建议

### 适合场景
- ✅ 论文有多个核心实验结果（如对比图、消融实验）
- ✅ 需要同时展示方法架构图和结果图
- ✅ 会议海报，需要在一屏中吸引关注
- ✅ 图片数量 >= 2 张的论文

### 推荐图片类型组合
1. **实验论文**: 结果对比图 + 性能曲线图
2. **方法论文**: 架构图 + 流程图
3. **应用论文**: 应用效果图 + 性能对比图
4. **理论论文**: 算法流程图 + 可视化结果图

## 🚀 如何使用

### Web UI (Gradio)

1. 启动 Gradio:
   ```bash
   python webui_gradio.py
   ```

2. 选择模板: **"学术会议-横向16:9 (academic_panels_landscape)"**

3. 上传 PDF，等待生成

4. 检查左侧 Key Figures 面板，应显示 2 张图片

### 命令行 (CLI)

```bash
python main.py input.pdf \
  --template academic_panels_landscape \
  --enable-vision \
  --max-images 15
```

## 🔍 技术细节

### Jinja2 列表操作

由于 Jinja2 模板中列表是不可变的，我们使用以下技巧：

1. **列表切片**: `sorted_results[:2]` 获取前 2 个元素
2. **列表过滤**: `all_sorted | reject('in', sorted_results)` 排除已选图片
3. **列表拼接**: `sorted_results + (...)` 合并两个列表

### 后备兼容性

保留了 `poster.main_figure_path` 的后备方案，确保与旧数据格式兼容：

```jinja2
{% elif poster.main_figure_path %}
    <div class="figure-container">
        <img src="images/{{ poster.main_figure_path }}" alt="Main Figure" class="figure-img">
        <div class="figure-caption">{{ poster.main_figure_caption }}</div>
    </div>
{% endif %}
```

## 🐛 已知问题

### 图片过多时
- 如果论文图片数量非常多（> 20 张），建议调整 `--max-images` 参数
- VLM 分析会过滤掉低优先级和无关图片（如期刊封面）

### 图片过少时
- 如果论文图片 < 2 张，会显示所有可用图片
- 如果只有 1 张图片，Key Figures 面板只显示 1 张（符合预期）
- 如果没有图片，面板保持空白或显示后备图片

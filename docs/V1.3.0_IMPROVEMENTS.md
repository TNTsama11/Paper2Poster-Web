### 2. 📐 **公式提取与展示** - 支持LaTeX/文本公式

**问题**: 只提取图片格式的公式，文本和LaTeX格式的公式被忽略

**解决方案**:

#### 2.1 新增公式提取器
```python
# src/harvester.py
class PDFHarvester:
    def extract_equations(self, text: str) -> List[Dict]:
        """
        从文本中提取公式，支持多种格式：
        1. 行内公式: $...$
        2. 行间公式: $$...$$
        3. LaTeX equation: \begin{equation}...\end{equation}
        4. 数学符号密集行（启发式）
        """
        equations = []
        
        # 模式1: $...$ 
        inline_pattern = r'\$([^\$]+)\$'
        
        # 模式2: $$...$$ 
        display_pattern = r'\$\$([^\$]+)\$\$'
        
        # 模式3: \begin{equation}...\end{equation}
        equation_pattern = r'\\begin\{equation\}(.*?)\\end\{equation\}'
        
        # 模式4: 数学符号密集行
        math_symbols = r'[∑∏∫∂∇≈≠≤≥±×÷√∞∈∉⊂⊃∪∩α-ωΑ-Ω]'
        
        return equations[:20]  # 最多20个
```

## 📊 支持的公式格式

### 1. 行内公式 `$...$`
```latex
The loss function is defined as $L = \sum_{i=1}^n (y_i - \hat{y}_i)^2$
```
**提取结果**: 
```
L = \sum_{i=1}^n (y_i - \hat{y}_i)^2
```

### 2. 行间公式 `$$...$$`
```latex
$$
E = mc^2
$$
```
**提取结果**:
```
E = mc^2
```

### 3. LaTeX equation 环境
```latex
\begin{equation}
\nabla \times \mathbf{E} = -\frac{\partial \mathbf{B}}{\partial t}
\end{equation}
```
**提取结果**:
```
\nabla \times \mathbf{E} = -\frac{\partial \mathbf{B}}{\partial t}
```

### 4. 数学符号密集文本（启发式）
```
Training objective: minimize ∑ᵢ₌₁ⁿ ‖xᵢ - x̂ᵢ‖²
```
**提取结果**:
```
minimize ∑ᵢ₌₁ⁿ ‖xᵢ - x̂ᵢ‖²
```

## 🔧 故障排查

### 问题1: 公式未提取
**检查**:
```bash
# 查看提取的公式数量
tail -20 output/poster.log | grep "提取到"

# 如果显示 "提取到 0 个公式"
```

**可能原因**:
1. PDF 中公式是图片格式（会显示在图片中）
2. 公式格式不规范（不是 $...$ 或 $$...$$）
3. 数学符号太少，未达到阈值

**解决**:
- 图片格式公式：已自动提取为图片
- 调低阈值：修改 `math_symbols` 匹配数量

### 问题2: 公式显示乱码
**原因**: LaTeX 公式未渲染

**解决**: 
- 当前版本显示原始 LaTeX 代码
- 未来版本可集成 MathJax 或 KaTeX 渲染

### 问题3: 内容仍然太少
**检查**: 使用的模型是否足够强

**建议**:
- GPT-4 或 GPT-4-turbo：内容最充实
- GPT-3.5：可能仍然简短
- 本地模型：视能力而定

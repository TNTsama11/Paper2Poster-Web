# v1.3.7 Abstract 字数控制优化

## 问题反馈

用户反馈：
1. **Abstract 内容不完整** - 即使优化了CSS，仍然可能出现截断
2. **需要字数控制** - 希望能够控制 Abstract 的长度
3. **Streamlit 不需要** - 放弃 Streamlit 界面，专注 Gradio

---

## 解决方案

### 1. LLM 生成时控制字数 ✅

**核心思路**: 在源头控制，让 LLM 生成时就精简到指定字数

#### 修改 System Prompt

```python
def _build_system_prompt(self, abstract_max_words: int = 130) -> str:
    return f"""你是一位专业的学术海报设计助手...
    
**重要要求**：
1. **Abstract**: 必须精简为 **{abstract_max_words}字以内** 的简洁摘要
   - 字数限制：严格控制在 {abstract_max_words}字以内（中文字符）
   - 内容要求：核心创新点 + 主要方法 + 关键结果
   - 如果原文摘要过长，务必重新组织语言，删减次要信息
...
"""
```

**优势**:
- ✅ 从源头控制，避免生成过长内容
- ✅ LLM 会主动精简和重新组织语言
- ✅ 保证关键信息完整

## 字数建议

### 按模板选择

| 模板 | 推荐字数 | 原因 |
|------|---------|------|
| **横版16:9 (landscape)** | 130字 ⭐ | 左侧栏空间有限 |
| **纵向会议 (panels)** | 150字 | 横跨全宽，空间充足 |
| **多图网格 (multi_figure)** | 120字 | 图片多，文字少 |
| **经典三栏 (simple)** | 140字 | 标准布局 |

### 按论文类型选择

| 论文类型 | 推荐字数 | 说明 |
|---------|---------|------|
| **理论型** | 150-180字 | 需要解释复杂概念 |
| **应用型** | 120-150字 | 方法和结果为主 |
| **实验型** | 130-160字 | 平衡方法和结果 |
| **综述型** | 100-130字 | 概括性强 |

### 按使用场景选择

| 场景 | 推荐字数 | 原因 |
|------|---------|------|
| **在线会议分享** | 100-130字 | 屏幕小，宜简洁 |
| **实体海报** | 150-180字 | 可近距离阅读 |
| **快速浏览** | 80-100字 | 抓住核心即可 |
| **详细展示** | 180-250字 | 完整传达信息 |

## 技术细节

### Prompt 关键部分

```
**Abstract 生成规则**：
- 字数限制：严格控制在 {abstract_max_words}字以内（中文字符）
- 内容要求：核心创新点 + 主要方法 + 关键结果
- 风格：简洁、清晰、突出重点
- 如果原文摘要过长，务必重新组织语言，删减次要信息
```

**强调点**:
1. "严格控制" - 让 LLM 注意字数限制
2. "重新组织语言" - 不是简单截断，而是重新表述
3. "删减次要信息" - 保留核心，去除细节

## 常见问题

### Q1: 为什么不在CSS层面截断？

**A**: CSS截断会导致：
- 句子被截断，语义不完整
- 可能截断在关键信息处
- 用户体验差

**LLM控制的优势**:
- 语义完整
- 主动精简，保留核心
- 更符合阅读习惯

### Q2: 130字够用吗？

**A**: 对于大多数场景够用：
- 海报重点是视觉和要点，不是详细叙述
- Abstract 太长会影响排版
- 详细内容在各个 section 中展开

如果论文特别复杂，可以调整到 150-180字。

### Q3: 字数是中文字符还是英文单词？

**A**: 
- **中文论文**: 中文字符数
- **英文论文**: LLM 会自然理解为英文单词数

Prompt 中明确说明 "中文字符"，对于中文论文更准确。

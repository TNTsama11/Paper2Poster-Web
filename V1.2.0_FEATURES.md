# Paper2Poster-Web v1.2.0 新功能总结

## 🎉 重大更新：视觉分析 + 多图海报

### 问题背景

**用户反馈**: "模板内容太空，很多空白区域。"

**根本原因**:
1. 只显示一张主图，其他图片浪费
2. 大量空白区域没有利用
3. 图片放置位置固定，不够智能

### 解决方案

✨ **引入视觉模型分析图片** + **多图海报模板**

---

## 🌟 核心功能

### 1. 视觉模型集成

使用 VLM（视觉语言模型）分析 PDF 中的每张图片：

- 📊 **图片类型识别**
  - architecture (架构图)
  - flowchart (流程图)
  - result (结果图) ⭐
  - chart (图表)
  - table (表格)
  - general (一般)

- 📝 **自动生成描述**
  - 每张图片一句话说明
  - 准确反映图片内容

- 📍 **智能放置建议**
  - introduction (引言区)
  - methods (方法区)
  - results (结果区)
  - any (任意位置)

### 2. 多图海报模板

**新增3个专业模板:**

#### A. multi_figure_grid.html

```
优化前（simple_grid.html）:
┌─────────┬─────────┬─────────┐
│  Text   │ 1 Image │  Text   │
│         │  大量空白  │         │
│  Text   │  大量空白  │  Text   │
└─────────┴─────────┴─────────┘

优化后（multi_figure_grid.html）:
┌────────────────────────────┐
│    Abstract (横跨全宽)       │
├────────┬──────────┬────────┤
│ Text   │  Image1  │  Text  │
│ Image2 │ (主图大)  │ Image4 │
│ Text   │  Image3  │  Text  │
└────────┴──────────┴────────┘
```

**特点**:
- ✅ 显示多张图片（最多6张）
- ✅ 摘要横跨全宽
- ✅ 主图居中且大
- ✅ 其他图片智能分布
- ✅ 充分利用空间
- ✅ 内容饱满不空

#### B. academic_panels.html ⭐ (最新)

借鉴 [Paper2Poster 项目](https://github.com/camel-ai/paper-to-poster) 的设计思路：

```
优化: 学术会议风格面板布局
┌─────────────────────────────┐
│    Abstract (横跨全宽)       │
├─────┬─────────┬─────────────┤
│Panel│  Panel  │    Panel    │
│  1  │   2     │      3      │
│     │ (Large) │   (Large)   │
├─────┴─────────┴─────────────┤
│ Panel 4 (Wide) │   Panel 5  │
├────────────────┴────────────┤
│  Panel 6  │ Panel 7 │ Panel │
└───────────┴─────────┴───────┘
```

**特点**:
- ✅ 9个独立面板布局
- ✅ 灵活的面板尺寸（large, wide, span）
- ✅ 学术会议海报标准
- ✅ 专业视觉效果
- ✅ 最高空间利用率（95%）
- ✅ 类似 LaTeX Beamer 海报

## 🚀 使用方法

### 方式 1: Web UI (最简单)

1. 启动界面:
   ```bash
   python webui_gradio.py
   ```

2. 配置：
   - ☑️ 启用视觉分析
   - ☑️ 使用多图模板
   - 视觉模型: 留空（自动推荐）

3. 上传 PDF，生成！

### 方式 2: 命令行

```bash
python main.py input/paper.pdf \
    --enable-vision \
    --vision-model gpt-4o-mini \
    -t multi_figure_grid.html
```

### 方式 3: 环境变量

```env
ENABLE_VISION_ANALYSIS=true
VISION_MODEL=gpt-4o-mini
```

## 🔬 技术细节

### 图片分析流程

```python
1. PDF解析 → 提取所有图片
2. 视觉模型 → 分析每张图片
   - 识别类型
   - 生成描述
   - 建议位置
3. 排序优先级 → result > architecture > flowchart > ...
4. 限制数量 → 最多保留6张
5. 渲染模板 → 智能放置图片
```

### 降级机制

```python
if 视觉分析失败:
    → 使用简单分析（基于文件名）
    → 仍然可以生成海报
    → 不会中断流程
```

### 性能优化

- 并行分析图片（如果API支持）
- 限制分析数量（默认6张）
- 智能缓存机制
- 自动模型推荐

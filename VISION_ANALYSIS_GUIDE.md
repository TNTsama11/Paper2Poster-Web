# 视觉分析功能使用指南

## 🎨 功能简介

Paper2Poster-Web v1.2.0 新增了**视觉模型分析**功能，可以：

✅ **智能分析图片内容** - 自动识别图片类型（架构图、结果图、流程图等）  
✅ **生成图片描述** - 为每张图片生成准确的说明文字  
✅ **智能放置图片** - 根据图片类型自动放置到海报合适位置  
✅ **多图片支持** - 支持在海报中展示多张图片，内容更丰富  
✅ **解决空白问题** - 充分利用海报空间，不再空荡荡

## 🌟 为什么需要视觉分析？

### 之前的问题

- ❌ 只显示一张主图，其他图片浪费
- ❌ 海报大片空白，内容稀疏
- ❌ 图片放置位置固定，不够智能
- ❌ 图片描述完全依赖LLM猜测

### 现在的优势

- ✅ 显示多张图片（最多6张）
- ✅ 内容丰富，充实饱满
- ✅ 图片智能分类和放置
- ✅ 每张图片都有准确描述

## 🚀 快速开始

### 方式 1: Web UI (推荐)

1. 启动 Gradio UI:
   ```bash
   python webui_gradio.py
   ```

2. 在界面中配置：
   - ☑️ 勾选 **"启用视觉分析"**
   - ☑️ 勾选 **"使用多图模板"**
   - 视觉模型留空（自动推荐）或指定（如 `gpt-4o-mini`）

3. 上传 PDF，点击生成

### 方式 2: 命令行

```bash
python main.py input/paper.pdf \
    --enable-vision \
    --vision-model gpt-4o-mini \
    -t multi_figure_grid.html
```

### 方式 3: 环境变量配置

在 `.env` 文件中添加：

```env
# 启用视觉分析
ENABLE_VISION_ANALYSIS=true
VISION_MODEL=gpt-4o-mini
```

然后正常运行：
```bash
python main.py input/paper.pdf
```

## 🎯 图片分析类型

视觉模型会将图片分为以下类型：

### 1. architecture (架构图)
- 系统架构图
- 模型结构图
- 网络结构图
- **放置位置**: Methods 部分

### 2. flowchart (流程图)
- 算法流程图
- 处理流程
- 工作流程
- **放置位置**: Methods 部分

### 3. result (结果图)
- 实验结果
- 性能对比
- 效果展示
- **放置位置**: Results 部分 (优先级最高)

### 4. chart (图表)
- 柱状图
- 折线图
- 饼图
- **放置位置**: Results 部分

### 5. table (表格)
- 数据表格
- 对比表格
- **放置位置**: Results 部分

### 6. general (一般图片)
- 其他类型
- **放置位置**: 任意

## 📐 多图模板布局

新的 `multi_figure_grid.html` 模板采用智能布局：

```
┌─────────────────────────────────────────────┐
│            Title / Authors                  │
├───────────────────────────────────────────┬─┤
│         Abstract (横跨全宽)                 │
├──────────┬───────────────┬────────────────┤
│  Introduction│  Main Figure  │   Results    │
│             │               │              │
│  [图片1]    │   (最大)      │   [图片4]   │
│             │               │              │
│  Methods    │  [图片2-3]    │  Conclusion  │
│             │               │              │
│             │               │  References  │
└─────────────┴───────────────┴──────────────┘
```

**特点:**
- 摘要横跨全宽
- 主图在中央，尺寸最大
- 其他图片根据类型智能放置
- 充分利用空间，不留空白

## 🔧 高级配置

### 自定义分析数量

默认最多分析 6 张图片，可以修改：

```python
# 在代码中
analyzed_figures = vision_analyzer.analyze_images(
    images_dir,
    image_manifest,
    max_images=10  # 改为 10 张
)
```

### 禁用视觉分析（降级模式）

如果不需要视觉分析，系统会自动降级为简单分析：

```python
# 基于文件名顺序简单推测
# img_01.png, img_02.png -> Methods
# img_03.png, img_04.png -> Results
# ...
```

### 混合使用

- 文本 LLM: 使用便宜的模型 (GPT-3.5, Qwen2.5-7B)
- 视觉 VLM: 使用视觉模型 (gpt-4o-mini, Qwen2-VL)

```env
OPENAI_MODEL=Qwen/Qwen2.5-7B-Instruct
VISION_MODEL=Qwen/Qwen2-VL-7B-Instruct
```

## 🐛 故障排查

### 问题 1: 视觉分析失败

**症状**: 显示 "图片分析失败"

**原因**:
- 视觉模型不支持
- API 余额不足
- 图片格式问题

**解决方法**:
1. 使用支持的视觉模型
2. 检查 API 余额
3. 系统会自动降级到简单分析

### 问题 2: 图片不显示

**症状**: 海报中图片显示空白

**原因**:
- 图片路径错误
- 图片文件损坏

**解决方法**:
1. 检查 `output/images/` 目录
2. 确认图片文件存在
3. 查看 HTML 源代码检查路径

### 问题 3: 成本过高

**原因**: 使用了昂贵的视觉模型

**解决方法**:
- 使用 `gpt-4o-mini` 而不是 `gpt-4o`
- 使用 SiliconFlow 的 Qwen2-VL 模型
- 减少分析的图片数量
